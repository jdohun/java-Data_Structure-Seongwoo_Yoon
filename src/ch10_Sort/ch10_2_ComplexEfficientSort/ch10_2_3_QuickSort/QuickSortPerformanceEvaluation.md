# 퀵 정렬: 성능평가(최선의 경우)

## Best Case: `중간 값`을 pivot으로 선택한 경우

### 8개의 data 가 있을 경우

#### 분할

- 8 -> 4 -> 2 -> 1
- 총 3번의 분할을 통해 비교 연산이 진행된다.
  = log 8 = 3
- n개의 경우 log n 번의 분할
-

#### 비교

- low 와 high 가 교차되기 전까지 pivot 과 계속 비교하기 때문에
  매 차수마다 대략 n 번 비교한다.

#### 따라서 최선의 빅 오 = O(n log n)

# 퀵 정렬: 성능평가(최악의 경우)

## Worst Case: pivot을 선택했으나 `두 그룹으로 나뉘어지지 않는 `경우

- pivot이 한쪽에 치우쳐져 n개의 원소가 모두 pivot이 된다면
  => 그룹이 나뉘는 횟수가 약 n번
- 매 단계별 비교 연산의 횟수가 n번

#### 따라서 최악의 빅 오 = O(n^2)
